## 一、现状问题

### 问题1：推荐同质化
当前50个专业的分数分布过于集中（6-8分之间），导致无论用户填什么条件，Top 5 经常重叠。根本原因：四个维度的数据是人工估算，精度不够。

### 问题2：热点缺失
"窗口期"维度目前是静态标签（平稳/上升/爆发/下行），不能反映：
- 国家政策风向（十四五、新质生产力）
- 当前就业市场需求（岗位数量、薪资水平）
- 人才供需缺口（培养人数 vs 市场需求）

### 问题3：个性化不足
现在所有人的权重都一样（AI 35% + 窗口 30% + 家庭 20% + 人格 15%），没有根据用户特征动态调整。

---

## 二、升级方案

### 方案A：不依赖数据库（可立即实现）

**A1. 增加"政策热度分"维度**

新增第5个维度：`policy_score`（政策导向分，1-10）

```
政策热度 10分：芯片/半导体、新能源汽车、低空经济、人工智能
政策热度 8分：生物医药、新材料、量子计算、航空航天
政策热度 6分：数字经济、绿色能源、高端制造
政策热度 4分：一般工科、金融、法学（稳定但无政策加成）
政策热度 2分：传统行业、人文社科
```

新权重：AI生存 30% + 窗口期 25% + 家庭 20% + 人格 15% + **政策 10%**

**A2. 动态权重（根据用户画像调整）**

| 用户特征 | 权重调整 |
|---------|---------|
| 家庭风险偏好高（risk=3） | 政策热度权重 +5%，稳定性权重 -5% |
| 家庭经济支撑强（economic=3） | 研究型专业加分 |
| 分数 700+（高分） | 解锁高门槛专业，顶级院校溢价 |
| 分数 <550（普通） | 强化"就业快/稳定"权重 |
| 父母在医疗行业 | 医学类专业额外 +1 分 |

**A3. 增加专业分化**

把50个专业的分差拉大，让结果更有区分度。例如：
- 人工智能：AI生存分从 8 升到 9.5
- 人力资源管理：AI生存分从 5 降到 3.5
- 低空经济/无人机：新增专业，窗口分 9.5

### 方案B：依赖数据库（需要1-2个月准备数据）

**数据表设计**

```
专业表 (majors)
├── 基础信息：名称、学制、学科门类
├── AI生存分（每半年更新一次）
├── 就业率（来源：教育部就业质量报告）
├── 平均薪资（来源：麦可思/Boss直聘）
├── 政策热度（人工打分，每季度更新）
└── 录取分数线（来源：阳光高考，分省份）

职业表 (occupations)
├── 职业名称
├── 关联专业（多对多）
├── 招聘需求指数（来源：招聘网站API）
└── 薪资区间（P25/P50/P75）

政策表 (policies)
├── 政策名称
├── 发布时间
├── 关联专业
└── 政策强度评级
```

**数据来源**

| 数据 | 来源 | 更新频率 |
|------|------|---------|
| 就业率、薪资 | 教育部年度就业质量报告（公开PDF） | 每年1次 |
| 录取分数线 | 阳光高考网、各省招生办 | 每年1次 |
| 政策热度 | 人工整理国家文件 | 每季度 |
| 招聘热度 | Boss直聘/猎聘（需申请API或爬虫） | 每月 |
